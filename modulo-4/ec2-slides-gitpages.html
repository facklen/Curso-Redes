O que é o Amazon EC2?

Computador remoto

  • VM na nuvem: liga, acessa, instala, desliga
  • Sem depender de hardware local
  • Paga pelo uso (tempo + armazenamento)

Analogia

  • Como o VirtualBox — mas nos data centers da AWS
  • Você escolhe o “tamanho” do computador
  • Quem entra passa pelo “porteiro” (Security Group)

VM local × EC2

  • Local: usa CPU/RAM/Disco do seu PC
  • Nuvem: recursos dos data centers AWS
  • Acesso remoto pela Internet (SSH)
  • Escalonamento fácil (mudar tipo de instância)

Conceitos fundamentais

  • AMI: imagem do sistema
  • Tipo: CPU/RAM
  • EBS: disco virtual
  • SG: firewall (portas)
  • VPC: rede
  • Chave .pem: acesso por SSH

Lançando uma instância — visão geral

  • Escolha a AMI (ex.: Ubuntu)
  • Selecione o tipo (ex.: t2.micro)
  • Crie/seleciona a chave .pem
  • Defina o Security Group (porta 22)
  • Lance e anote o IP público

Boas práticas

  • Porta 22 limitada a My IP
  • chmod 400 chave.pem
  • Encerrar recursos ao final

Passo a passo no Console

  1. EC2 → Launch instances
  2. Nome + AMI + tipo t2.micro
  3. Par de chaves (.pem)
  4. Security Group (SSH 22 / My IP)
  5. Launch → copiar IP público

Usuários comuns:

  • ubuntu@IP (Ubuntu)
  • ec2-user@IP (Amazon Linux 2)

Conexão via SSH

chmod 400 MINHA-CHAVE.pem
ssh -i MINHA-CHAVE.pem ubuntu@IP-PUBLICO

Verifique SG porta 22 e usuário correto.

Ciclo de vida da instância

  • running: ligada
  • stopped: desligada (EBS persiste)
  • terminated: removida

Dica: parar ≠ encerrar

  • Parar: paga armazenamento do EBS
  • Encerrar: instância removida (volume pode ser apagado)

Segurança essencial

  • Proteja a chave .pem
  • Evite 0.0.0.0/0 em produção
  • Use usuários padrão + sudo
  • Rotacione chaves quando necessário
  • Feche portas não usadas
  • Atualize pacotes do SO
